# Expert-to-Skill: 전문가의 암묵지를 실행 가능한 AI 스킬로 변환하는 시스템

> 인스타, 유튜브, 쓰레드의 전문가 노하우를 관찰하고, 이를 SKILL.md로 코드화하여, Claude Code + MCP로 그 전문가처럼 작업하게 만드는 시스템의 설계 철학.

---

## 1. 비전: "전문가의 뇌를 복제하는 것이 아니라, 작업 방식을 복제한다"

### 핵심 관찰

세상에는 각 분야의 고수들이 있습니다:
- PPT를 압도적으로 잘 만드는 사람
- 쇼츠/릴스를 중독성 있게 편집하는 사람
- 20년차 대본작가가 쓰는 스토리텔링 구조
- 데이터를 한눈에 보이게 시각화하는 사람
- 노션을 지식 시스템으로 설계하는 사람

이들의 노하우는 어디에 있는가?
- 유튜브 튜토리얼 영상
- 인스타그램/쓰레드의 팁 게시물
- 온라인 강의와 워크숍
- 그들이 만든 결과물 자체

**문제: 이 노하우는 "알려진" 상태지만 "실행 가능한" 상태가 아닙니다.**

사람이 이 노하우를 배우려면 수십 시간 학습 → 수백 시간 연습이 필요합니다. 하지만 그 노하우의 본질이 "이런 상황에서 이런 판단을 하고 이런 순서로 작업한다"는 **절차적 지식(procedural knowledge)**이라면, 이것은 SKILL.md로 코드화할 수 있습니다.

### 시스템의 공식

```
전문가의 콘텐츠 (유튜브/인스타/쓰레드)
    │
    ▼ [관찰 + 추출]
전문가의 작업 패턴 (암묵지 → 형식지)
    │
    ▼ [코드화]
SKILL.md (선언적 워크플로우 명세)
    │
    ▼ [실행]
Claude Code + MCP 도구 → 결과물
```

## 2. 현재 시스템이 이미 증명한 것

이 프로젝트(newVideoGen)는 위 공식의 **첫 번째 성공적인 인스턴스**입니다.

### 이미 일어난 일

```
관찰 대상: "교육 영상을 잘 만드는 전문가의 패턴"
    │
    ├── 대본 구조: 도입(15%) → 전개(60%) → 절정(15%) → 마무리(10%)
    ├── 화면 설계: 의미에 따라 레이아웃을 선택하는 판단 기준
    ├── 모션 연출: 요소 등장 타이밍과 자막 동기화 기법
    ├── 시각 어휘: 70종 REF 레퍼런스 패턴 라이브러리
    └── 에셋 활용: 이미지를 내용과 매칭하는 방식
    │
    ▼ [SKILL.md로 코드화]
    │
    ├── /reat-script  ← 대본작가의 구조 설계 노하우
    ├── /reat-layout  ← 영상 디자이너의 레이아웃 판단 기준
    ├── /reat-assets  ← 에셋 큐레이터의 태깅/매칭 방법론
    └── /reat-render  ← 영상 편집자의 렌더링 워크플로우
    │
    ▼ [Claude Code가 실행]
    │
    결과: 주제 텍스트 하나로 교육 영상 자동 생성
```

**이것이 작동한다는 것은 이미 증명되었습니다.** 문제는 이 패턴을 다른 도메인으로 일반화하는 것입니다.

## 3. Expert-to-Skill 파이프라인: 4단계 프로세스

### Stage 1: 수집 (Capture)

전문가의 콘텐츠를 수집하고 구조화합니다.

```
입력 소스:
├── 유튜브 영상 → 자막 추출 + 화면 캡처 + 구조 분석
├── 인스타 게시물 → 이미지 분석 + 캡션 텍스트
├── 쓰레드 → 텍스트 스레드의 논리 구조
├── 온라인 강의 → 단계별 워크플로우
└── 결과물 자체 → 역분석 (어떤 원칙으로 만들었는가)
```

**수집의 핵심은 "무엇을 했는가"가 아니라 "왜 그렇게 했는가"를 추출하는 것입니다.**

예시 — PPT 고수의 유튜브 영상에서:
```
관찰: "이 사람은 항상 한 슬라이드에 하나의 메시지만 넣는다"
추출: Rule — 1 slide = 1 message. 두 개 이상이면 분리.

관찰: "숫자를 강조할 때 항상 180pt 이상의 폰트를 쓴다"
추출: Rule — 핵심 수치는 본문 폰트의 6배 이상.

관찰: "비교할 때 항상 좌우 분할 + 색상 대비를 쓴다"
추출: Rule — 비교 → Split layout + 좌측 gray/우측 accent.
```

### Stage 2: 정제 (Distill)

수집된 원시 패턴을 **판단 기준(decision criteria)**과 **실행 절차(procedure)**로 정제합니다.

```
원시 패턴:
  "이 사람은 도입부에서 항상 충격적인 숫자를 먼저 보여준다"

정제된 판단 기준:
  IF scene.position == "intro" AND content.has_statistics:
    THEN lead_with = "StatNumber" (size: xl, motion: popNumber)
    THEN follow_with = "Headline" (반문형 질문)

정제된 실행 절차:
  1. 대본에서 가장 임팩트 있는 숫자를 찾는다
  2. 그 숫자를 화면 중앙에 크게 배치한다
  3. 0.5초 후 질문형 헤드라인이 아래에서 올라온다
  4. 2초 후 맥락 설명 텍스트가 페이드인 된다
```

### Stage 3: 코드화 (Encode)

정제된 지식을 SKILL.md 형식으로 코드화합니다.

```markdown
# /expert-ppt-layout — PPT 고수 스타일 레이아웃

## 판단 기준

### 슬라이드 유형 결정
| 콘텐츠 특성 | 슬라이드 유형 | 핵심 규칙 |
|------------|------------|----------|
| 핵심 숫자 1개 | big-number | 180pt+, 화면 중앙, 보조 텍스트 최소화 |
| A vs B 비교 | split-compare | 좌우 분할, 색상 대비, 동일 구조 대칭 |
| 3~5개 나열 | icon-grid | 아이콘 + 한 줄 레이블, 균등 배치 |
| 인용문 | quote-fullscreen | 큰 따옴표 + 출처, 배경 어둡게 |
| 프로세스 | step-flow | 번호 + 화살표, 좌→우 진행 |

### 디자인 불변 규칙
1. 1 슬라이드 = 1 메시지 (예외 없음)
2. 텍스트 3줄 초과 금지 → 초과 시 슬라이드 분리
3. 여백 40% 이상 유지 → 빽빽하면 요소 제거
4. ...

## 워크플로우
1. 콘텐츠를 의미 단위로 분리 (1 메시지 = 1 슬라이드)
2. 각 단위의 유형 판단 (위 표 기준)
3. 유형에 맞는 레이아웃 생성
4. MCP 도구로 PPT 실제 생성
```

### Stage 4: 실행 (Execute)

SKILL.md + MCP 도구로 Claude Code가 실제 작업을 수행합니다.

```
/expert-ppt-layout "AI 시대의 교육 혁신" --style "minimalist"
    │
    ├── Claude가 SKILL.md를 읽고 판단 기준 습득
    ├── 주제를 분석하여 슬라이드 구조 설계
    ├── 각 슬라이드의 레이아웃 JSON 생성
    └── PPT MCP 도구로 실제 PPTX 파일 생성
```

## 4. 구체적 시나리오: 세 가지 Expert 스킬

### 시나리오 A: PPT 고수의 스킬

```
수집 대상: 인스타에서 10만 팔로워의 PPT 전문가
수집 내용: "발표 자료 디자인의 7가지 원칙" 게시물 시리즈

추출된 스킬:
  /expert-ppt-design
  ├── 판단: 콘텐츠 유형 → 슬라이드 유형 매핑
  ├── 규칙: 1슬라이드 1메시지, 여백 40%, 색상 3개 제한
  ├── 패턴: 표지→목차→본문→전환→마무리 흐름
  └── 스타일: 미니멀/볼드/기업/학술 변형

실행 체인:
  /expert-ppt-design "분기 실적 보고"
      → 슬라이드 구조 설계 (JSON)
      → PPT MCP로 PPTX 생성
      → 또는 Google Slides MCP로 프레젠테이션 생성
```

### 시나리오 B: 쇼츠 전문가의 스킬

```
수집 대상: 유튜브에서 쇼츠 편집 노하우 영상을 올리는 크리에이터
수집 내용: "첫 3초에 시청자를 잡는 법", "자막 타이밍의 비밀"

추출된 스킬:
  /expert-shorts-edit
  ├── 판단: 후킹 유형 선택 (충격 숫자/질문/반전/감정)
  ├── 규칙: 첫 1초 텍스트 등장, 3초마다 시각 전환, 60초 이내
  ├── 패턴: 후킹→긴장→전개→반전→CTA
  ├── 자막: 3단어씩 끊기, 핵심 단어 색상 강조, 위치 변주
  └── 모션: 줌인/줌아웃 리듬, 비트 동기화

실행 체인:
  /expert-shorts-edit "AI가 일자리를 없앨까?"
      → 대본 구조: 후킹(5초) → 본론(45초) → CTA(10초)
      → 자막 생성: 3단어 단위 분할 + 강조 토큰
      → 모션 설계: 3초 리듬 시각 전환
      → /reat-render로 세로형 영상 렌더링
```

### 시나리오 C: 베테랑 대본작가의 스킬

```
수집 대상: 쓰레드에서 스토리텔링 노하우를 공유하는 20년차 작가
수집 내용: "사람들이 끝까지 보는 영상의 대본 구조"

추출된 스킬:
  /expert-scriptwriting
  ├── 판단: 주제 유형별 최적 서사 구조 선택
  │   ├── 설명형: 문제 제기 → 개념 도입 → 사례 → 정리
  │   ├── 설득형: 충격 → 공감 → 논거 → 전환 → 행동 촉구
  │   └── 스토리형: 인물 → 갈등 → 시도 → 깨달음 → 변화
  ├── 규칙:
  │   ├── 한 문장 = 15~25자 (읽기 2~3초)
  │   ├── 추상 1 : 구체 3 비율
  │   ├── 30초마다 "전환어 후킹" 삽입
  │   └── 핵심 메시지 3회 변주 반복
  └── 톤: 친구체/강의체/스토리체 자동 선택

실행 체인:
  /expert-scriptwriting "AI 시대의 생존 전략" --structure persuasion
      → script.json 생성 (작가의 구조대로)
      → Notion MCP로 대본 페이지 작성
      → 또는 /reat-voice → /reat-layout → /reat-render로 영상화
```

### 핵심 분리

```
스킬 (SKILL.md)  = 전문가의 "뇌" (판단 기준 + 작업 방식)
Claude Code      = "신경계" (스킬을 이해하고 판단을 실행)
MCP 도구         = "손" (실제 도구를 조작)
```

이 분리가 왜 강력한가:
- **스킬은 도구에 독립적**: 같은 대본 작성 스킬로 Notion에 쓸 수도, Google Docs에 쓸 수도 있음
- **도구는 스킬에 독립적**: PPT MCP는 어떤 스킬이 호출하든 동일하게 작동
- **Claude Code가 중개**: 스킬의 판단을 도구의 동작으로 번역

## 6. SKILL.md의 해부학: 전문가 지식의 구조

### 전문가의 암묵지가 SKILL.md에서 어떻게 표현되는가

전문가의 지식은 크게 4가지 층위로 분해됩니다:

```
┌─────────────────────────────────────┐
│  Layer 1: 판단 기준 (Decision)       │
│  "이 상황에서는 이것을 선택한다"       │
│                                     │
│  예: intent가 compare이면            │
│      Split 레이아웃 + 색상 대비       │
├─────────────────────────────────────┤
│  Layer 2: 작업 절차 (Procedure)      │
│  "이 순서로 작업한다"                 │
│                                     │
│  예: 1. 콘텐츠 분석                  │
│      2. 구조 설계                    │
│      3. 시각 요소 배치               │
│      4. 모션 적용                    │
│      5. 후처리 보정                  │
├─────────────────────────────────────┤
│  Layer 3: 품질 규칙 (Constraints)    │
│  "이것은 반드시 / 절대"               │
│                                     │
│  예: 1슬라이드 1메시지 (불변)         │
│      텍스트 3줄 초과 금지             │
│      여백 40% 이상                   │
├─────────────────────────────────────┤
│  Layer 4: 레퍼런스 라이브러리 (Ref)   │
│  "이런 패턴들이 있다"                 │
│                                     │
│  예: REF-001 히어로 레이아웃          │
│      REF-002 좌우 비교               │
│      REF-003 3열 그리드              │
│      (= 전문가가 실제 만든 결과물들)   │
└─────────────────────────────────────┘
```

### 현재 /reat-layout이 이 4개 층위를 모두 갖고 있음

```
Layer 1 (판단): intent → 아키타입(A~Z) 매핑 테이블
Layer 2 (절차): 1.로드 → 2.분석 → 3.REF매칭 → 4.생성 → 5.후처리
Layer 3 (규칙): SceneRoot alignItems:center, gap max 28, 단일카드 maxWidth 520
Layer 4 (레퍼런스): 70종 REF 라이브러리 (docs/layout-reference-library.md)
```

## 7. Knowledge Capture: 전문가 노하우를 어떻게 추출하는가

### 수집 채널별 추출 방법

#### YouTube 영상 → 절차적 지식 추출

```
1. 자막 추출 (SRT)
2. Claude가 자막을 분석하여 구조화:
   - "먼저 ~합니다" → Step 1
   - "다음으로 ~합니다" → Step 2
   - "절대 ~하지 마세요" → Constraint
   - "이게 핵심인데요" → Key Decision
3. 영상 스크린샷 캡처 → 결과물 역분석
4. Before/After 비교 → 규칙 추출
```

#### Instagram 게시물 → 시각적 규칙 추출

```
1. 이미지 분석 (Claude 멀티모달)
   - 레이아웃 패턴 (그리드, 여백, 정렬)
   - 색상 팔레트 (주색, 보조색, 배경)
   - 타이포그래피 (크기 비율, 계층)
2. 캡션 텍스트 → 설명된 원칙 추출
3. 캐러셀 슬라이드 → 순서/흐름 패턴
```

#### Threads 텍스트 → 판단 프레임워크 추출

```
1. 스레드 전체 텍스트 수집
2. 구조 분석:
   - "~할 때는 ~하세요" → Decision Rule
   - "저는 항상 ~합니다" → Default Behavior
   - "흔한 실수: ~" → Anti-pattern
   - "비결은 ~입니다" → Key Principle
3. 판단 트리 구성 → SKILL.md의 판단 기준 섹션
```

## 8. 실행 계층: MCP가 만드는 무한 확장

### MCP = 스킬의 "손"

스킬이 "무엇을 판단하는가"를 정의한다면, MCP는 "무엇을 조작할 수 있는가"를 결정합니다.

```
스킬이 결정:        MCP가 실행:
──────────────      ──────────────
"표지 슬라이드를     PPT MCP: addSlide(layout: "title")
 만들어야 한다"      Google Slides MCP: createSlide(...)

"이 내용을           Notion MCP: createPage(...)
 문서로 작성한다"     Google Docs MCP: insertText(...)

"이 데이터를          Excel MCP: createChart(...)
 차트로 보여준다"     Google Sheets MCP: addChart(...)

"이 결과를            Slack MCP: postMessage(...)
 팀에 공유한다"       GitHub MCP: createIssue(...)
```

### 현재 사용 가능한 MCP 도구들

이 프로젝트에 이미 연결된 MCP:
- **Notion MCP**: 페이지 생성/편집, 데이터베이스, 댓글
- **GitHub MCP**: 파일 CRUD, PR, 이슈, 릴리스
- **Playwright**: 브라우저 제어, 스크린샷, 폼 입력
- **Hugging Face**: 모델 검색, 문서 조회

추가 연결 가능한 MCP:
- PPT/PPTX 생성 (python-pptx 기반 커스텀 MCP)
- Google Slides / Google Sheets
- Figma
- Slack
- 이메일

### 같은 스킬, 다른 출력

```
/expert-scriptwriting "AI 시대의 생존 전략"
    │
    ├──→ --output notion  → Notion MCP로 대본 페이지 작성
    ├──→ --output video   → /reat-voice → /reat-layout → /reat-render
    ├──→ --output ppt     → PPT MCP로 발표 자료 생성
    └──→ --output doc     → Word/PDF 문서 생성
```

**같은 전문가의 판단 기준이, 다른 도구로 다른 형태의 결과물을 만듭니다.**

## 10. 핵심 원리 요약

### 원리 1: 전문가의 가치는 "결과물"이 아니라 "판단 기준"에 있다

PPT를 잘 만드는 사람의 진짜 가치는 "예쁜 슬라이드"가 아니라 "이 내용에는 이 레이아웃이 맞다"는 판단력입니다. 이 판단 기준을 SKILL.md로 코드화하면, AI가 그 판단을 재현할 수 있습니다.

### 원리 2: SKILL.md는 "프로그램"이 아니라 "프로토콜"이다

스킬은 코드로 실행되는 것이 아니라, Claude Code가 읽고 이해하여 따르는 약속입니다. 그래서 자연어로 작성되고, 맥락에 따라 유연하게 적용됩니다. if/else가 아니라 judgment입니다.

### 원리 3: 스킬과 도구의 분리가 확장성의 핵심이다

"어떻게 판단하는가"(스킬)와 "무엇을 조작하는가"(MCP)가 분리되어 있기 때문에:
- 같은 스킬로 다른 도구를 조작할 수 있고 (PPT → Google Slides)
- 같은 도구를 다른 스킬이 사용할 수 있습니다 (Notion을 대본용/기획용으로)

### 원리 4: AI는 "실행"이 아니라 "번역"을 한다

Claude Code의 역할은 스킬(전문가의 판단 기준)을 도구(MCP 명령)로 번역하는 것입니다. 전문가가 "여기서 비교 레이아웃을 써야 해"라고 판단하면, Claude Code가 그것을 PPT의 `addSlide({layout: "two-column"})` 호출로 번역합니다.
